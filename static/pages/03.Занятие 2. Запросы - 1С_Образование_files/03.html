Запросы SQL через PHP

Рассмотрим подключение к базе данных через PHP. Запускаем Visual Studio Code и создаем папку «connect», в которой будут храниться 2 файла: «setting.php» и «index.php». В первый файл запишем код для настройки и подключения к базе.

<?php

$data = 'socialsite';

Комментарии обозначаются двумя слешами (//) и не считываются отладчиком.

Появится ошибка «Error», которую уже прописывали при недопуске к пользователю. Возвращаем настройки пароля, обновляем localhost, восстанавливая соединение.

Запросы через PDO

 

PDO – это модуль объекта данных, который определяет простой и согласованный интерфейс для доступа к базам данных в PHP. Каждый драйвер базы данных, в котором реализован этот интерфейс, может представить специфичную для базы данных функциональность в виде стандартных функций модуля. Но надо заметить, что сам по себе модуль PDO не позволяет манипулировать доступом к базе данных. Чтобы воспользоваться возможностями PDO, необходимо использовать соответствующий конкретной базе данных PDO драйвер.

Ранее мы писали код при помощи MySQL I, теперь попробуем написать запрос в базе данных посредством модуля PDO. Открываем Visual Studio Code  и переименовываем файл «index.php» в «index i.php». Создадим еще один файл «index PDO.php», в нем выполним запрос к базе данных. С помощью синтаксиса PDO подключимся к нашему серверу, написав наш драйвер и через параметры указав имя и пароль пользователя.

<?php

$username = 'root';

$passw = 'mysql';

 

$connectPDO = new PDO('mysql:host=localhost; dbname=socialsite', $username, $passw);

Следующим шагом будет написание SQL-строки в базе данных. Сделаем запрос на удаление данных, но перед этим изменим имя таблицы в базе данных с «users1» на просто «users» через операции (рис. 2.7). Возвращаемся к коду, прописываем запрос на удаление пользователя и при помощи глобальной функции «exec» передаем запрос.

$connectPDO = new PDO('mysql:host=localhost; dbname=socialsite', $username, $passw);

Подготовленные запросы

Для понимания разницы между простыми, прямыми и подготовленными запросами, нужно знать, что такое SQL-инъекция.

 

SQL-инъекция – это атака, направленная на веб-приложение, в ходе которой SQL-выражение вводится пользовательским способом путем простой конкатенации.

Что касается защиты и запросов, то атакующий может изменить логику выполнения SQL-запроса так, что он сможет получить доступ к некоторым базам. Поэтому с самого начала необходимо учиться правильному написанию кода.  Нужно понимать, что данные пользователя мы будем получать от пользователя с формы.

Откроем Visual Studio Code и продолжим писать код. В первую очередь напишем подготовленные запросы, задокументируем в комментарий кусочек кода.

<?php

$username = 'root';

$passw = 'mysql';

 

$connectPDO = new PDO('mysql:host=localhost; dbname=socialsite', $username, $passw);

 


/*

$query = "delete from users where name='ilya'";

$request = $connectPDO->exec($querry);

print ("Запросов было выполнено $request");

*/

Пишем комментарий «Подготовленные запросы» и начинаем писать их. В первую очередь мы должны получать данные как будто бы из формы. Для этого создадим переменные и заполним их. После чего приступим к написанию массива «arrayusers», он будет состоять из ключей.

С подготовленными запросами всегда работаем через массив, даже если передается всего один параметр.  

 

<?php

$username = 'root';

$passw = 'mysql';

 

$connectPDO = new PDO('mysql:host=localhost; dbname=socialsite', $username, $passw);

/*

$query = "delete from users where name='ilya'";

$request = $connectPDO->exec($querry);

print ("Запросов было выполнено $request");

*/

//Подготовленные запросы

 

$name = 'ilya';

$age = '35';

$login = 'user';

$password = '1234';

 

$arrayusers = [

    'Name' => $name,

    'Age' => $age,

    'Login' => $login,

    'Password' => $password

];

Теперь представим, что мы заполняем данные с формы и необходимо их отправить на нее. Создаем переменную «sqlquery» и в ней при помощи двоеточия прописываем параметры, наши ключи. В переменную «request» записываем обращение к экземпляру класса PDO, вызываем этот метод и в нем вызываем запрос. После обращаемся к нашему массиву через «execute».

 

Метод «prepare» - подготавливает запрос к выполнению и возвращает связанный с этим запросом объект.

Метод «execute» - запускает подготовленный запрос на выполнение.

Текст

$sqlquery = "insert users (name, age, login, password) value (:Name, :Age, :Login, :Password)";

$request = $connectPDO->prepare($sqlquery);

$request->execute($arrayusers);

Открываем в браузере наш localhost, обновляем страницу, видим, что код исчез. Заходим на phpMyAdmin в базу «socialsite», таблицу «users», переходим во вкладку «Обзор» и видим наши данные пользователя (рис. 2.14).
