Ответы на контрольные вопросы

Занятие 1

  1. Константа хранит в себе одно значение, которое редко изменяется во времени.
  2. В пользовательском режиме пользователь видит синоним.
  3. Чтобы отображать текстовые данные, используется тип "строка".
  4. Для хранения ограниченного набора значений используется "Перечисление".

Занятие 2

  1. Константа хранит в себе одно значение, которое редко изменяется во времени.
  2. В пользовательском режиме пользователь видит синоним.
  3. Чтобы отображать текстовые данные, используется тип "строка".
  4. Для хранения ограниченного набора значений используется "Перечисление".

Занятие 3

  1. Объект, позволяющий хранить данные о событиях в жизни фирмы/организации, называется документом.
  2. В модуле объекта.
  3. Обычная группа отличается от группы без отображения тем, что у нее отображается заголовок.
  4. Табличная часть нужна для того, чтобы хранить данные, имеющие одинаковую структуру, количество которых заведомо неизвестно. 

Занятие 4

  1. Схема бизнес-процесса начинается с точки старта.
  2. Чтобы реализовать проверку условия на карте маршрута бизнес-процесса, используется точка условия.
  3. Бизнес-процесс не может существовать без задач.

Занятие 5

  1. В регистр накопления данные записываются из документа по правилу, заданному в "ОбработкеПроведения".
  2. В регистр накопления данные попадают в момент проведения документа.
  3. У регистра накопления ресурс может быть только числовым типом данных.
  4. Определить событие, а также описать его автоматическую обработку можно с помощью точки обработки.

Занятие 6

  1. В виртуальной таблице "Остатки" хранятся данные об остатках.
  2. К виртуальной таблице можно обратиться с помощью конструкции "РегистрыНакопления.НазваниеРегистра.Остатки()".
  3. Для того чтобы найти данные в таблице, можно использовать метод "Найти()".
  4. Для того чтобы проверить, была ли найдена информация в результате выполнения какого-то алгоритма, используется тип "Неопределено".

Занятие 7

  1. Для того чтобы методы, описанные в серверном общем модуле, можно было вызывать с клиента, необходимо в палитре свойств этого модуля поставить галочку "Вызов сервера".
  2. Для того чтобы процедуру можно было вызвать из другого модуля, нужно указать у нее ключевое слово "Экспорт".
  3. Если необходимо получить или обработать данные, которые находятся в базе, и пользователь не видит их перед собой, то такой алгоритм должен выполняться на сервере. 

Занятие 8

  1. Чтобы процедура вызывалась в фоновом режиме, необходимо использовать объект "ФоновыеЗадания.Выполнить()".
  2. Чтобы заполнить командную панель самостоятельно, у нее нужно отключить свойство "Автозаполнение".
  3. У обычных групп отображается заголовок.

Занятие 9

  1. Чтобы измерение в регистре было обязательным к заполнению, необходимо поставить галочку "Запрет незаполненных значений" в его палитре свойств. 
  2. Чтобы описать логику движения данных из документа в регистр, можно использовать конструктор движения.
  3. Периодический регистр сведений нужен для того, чтобы можно было получать актуальные данные в рамках определенного периода.

Занятие 10

  1. С помощью типа "Файл" можно получить данные о прикрепляемом файле.
  2. В реквизите с типом "Хранилище значения" хранятся двоичные данные.
  3. Чтобы элемент формы выводился в виде картинки, у него нужно указать вид "Поле картинки". 

Занятие 11

  1. "СрезПоследних()" – это метод, благодаря которому можно получить последние актуальные данные из регистра сведений.
  2. С помощью метода "Получить()" можно узнать, какое значение хранится в константе.
  3. Функции могут возвращать значения.
  4. Структура представляет из себя пару "ключ – значение".

Занятие 12

  1. С помощью метода "Установить()" можно изменить значение константы.
  2. Метод "Оповестить()" сообщает формам, что произошло событие.
  3. С помощью метода "Сообщить()" можно вывести информацию пользователю.

Занятие 13

  1. В регистр накопления данные попадают из документов.
  2. С помощью метода "Итог()" можно посчитать сумму всех строк колонки таблицы.
  3. В регистр накопления данные попадают в момент проведения документа.

Занятие 14

  1. Параметр сеанса – это механизм, который позволяет понять, кто сейчас зашел в программу.
  2. Чтобы кнопка "Заполнить выражение" сработала, должны быть соблюдены два критерия:
  • тип данных поля и реквизита документа должны совпадать;
  • названия поля и реквизита документа должны совпадать.

Занятие 15

  1. Отчет – это объект конфигурации, который позволят анализировать и выводить данные в удобном для пользователя формате.
  2. Для выбора поля для отображения в отчете используется конструктор запроса.

Занятие 16

  1. Роль – это набор полномочий, который указывает, что может делать пользователь, а что – нет.
  2. Пользователь с правами на администрирование и администрирование данных обязательно должен быть в системе.
  3. Стиль – механизм, позволяющий настроить оформление программы. 

Занятие 17

  1. Параметр в запросе обозначается с помощью символа "&".
